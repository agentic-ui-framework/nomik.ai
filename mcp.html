Tools in. Tools out. One gate.

Nomik agents speak MCP natively: it's how they hold their tools internally, so it's also how you extend them. Tools in

Mount any MCP server. Live today.

The MCP ecosystem is the biggest tool catalog in AI: databases, warehouses, internal services, the servers you write yourself. Point a workspace at one and its tools show up in the roster's hands, risk-classed like everything else.

  • Any server works: the ones you find, the ones you write, the ones your vendors ship.
  • Same guardrails: mounted tools inherit the approval gates and the audit trail.
  • No raw secrets: agents act through scoped access, never your credentials in the clear.

One dispatch, one gate

Every tool walks through the same door.

Built-in tools, connectors, your MCP servers, tools an external agent lends over the SDK: one dispatch path, one risk model, one approval floor, one trail. There is no ungated side entrance.

Risk-classed at mount

Each tool declares a tier, from free reads to money and destructive. You can tighten any mounted tool's tier; you can't loosen the floor.

Approvals where they belong

A mounted tool that moves money or deletes data parks for a human, exactly like a built-in one. The protocol doesn't change the policy.

Indistinguishable in the trail

In the activity trace, your warehouse tool looks exactly like a native connector: who called it, with what, what it cost.

Tools out

Nomik speaks MCP back.

The protocol runs both directions. Nomik Desktop already ships as an MCP client and an MCP server, so other MCP-speaking apps can use its governed tools. And the workspace's own tool surface over MCP, for your IDE and your stack, is on the roadmap, told straight.

Why MCP

Extensibility without a trust hole.

Most platforms bolt tools on around the governance. On Nomik the protocol surface and the policy surface are the same surface, which is the point.

The standard, not a fork

MCP is the ecosystem's lingua franca. Anything written for it works here, and anything built here stays portable.
