In practice

A feel for the shape of it.

Clean REST, a familiar MCP config, and a one-row connector. Full details, auth, and error codes live in the docs.

REST · list a workspace's agents
# Authenticated with a bearer token
curl https://api.nomik.ai/v1/workspaces/$WS/agents \
  -H "Authorization: Bearer $TOKEN"

# → every response shares one envelope
{
  "success": true,
  "data": [
    { "id": "agt_8f2", "name": "Fin", "role": "finance" }
  ],
  "metadata": { "requestId": "req_3a9" }
}
MCP · give agents a tool you host
// Mount any MCP server into the workspace.
// Its tools join the ones agents already have.
{
  "mcpServers": {
    "warehouse": {
      "command": "npx",
      "args": ["-y", "@acme/warehouse-mcp"],
      "env": { "REGION": "eu" }
    }
  }
}
Connector · one row becomes a real integration
// No SDK to write. Declare it; the factory builds the actions,
// the vault holds the credential, and risk gates the writes.
export const acme = {
  kind: "acme",
  displayName: "Acme",
  category: "Operations",
  auth: "api_key",            // or "oauth"
  baseURL: "https://api.acme.com/v2",
  risk: "outbound",           // reads stay free; writes wait for approval
}

Nomik Local

The same SDK powers a local-first agent.

Nomik Local is a standalone agent that runs on your machine, with your model and your tools, safe by default. One connect and it plugs into a Nomik workspace through this very SDK: local hands, workspace brain, all under the human approval floor.

  • Safe by default: encrypted secrets, sandbox-first exec, risk-gated tools, an audit log, on at install.
  • Workspace-native: governed by the same gate, spend caps, RBAC, and activity trail your fleet already rides.
  • Bring your skills: import your existing OpenClaw skills, then version and score them.
Meet Nomik Local →

# a local agent, one command from your workspace
