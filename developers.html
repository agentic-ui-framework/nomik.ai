In practice

A feel for the shape of it.

Clean REST, a familiar MCP config, and a one-row connector. Full details, auth, and error codes live in the docs.

REST · list a workspace's agents
# Authenticated with a bearer token
curl https://api.nomik.ai/v1/workspaces/$WS/agents \
  -H "Authorization: Bearer $TOKEN"

# → every response shares one envelope
{
  "success": true,
  "data": [
    { "id": "agt_8f2", "name": "Fin", "role": "finance" }
  ],
  "metadata": { "requestId": "req_3a9" }
}
MCP · give agents a tool you host
// Mount any MCP server into the workspace.
// Its tools join the ones agents already have.
{
  "mcpServers": {
    "warehouse": {
      "command": "npx",
      "args": ["-y", "@acme/warehouse-mcp"],
      "env": { "REGION": "eu" }
    }
  }
}
Connector · one row becomes a real integration
// No SDK to write. Declare it; the factory builds the actions,
// the vault holds the credential, and risk gates the writes.
export const acme = {
  kind: "acme",
  displayName: "Acme",
  category: "Operations",
  auth: "api_key",            // or "oauth"
  baseURL: "https://api.acme.com/v2",
  risk: "outbound",           // reads stay free; writes wait for approval
}
On the roadmap

What we're building next, said honestly.

These aren't shipped yet. We'd rather tell you what's coming than dress up a mock as a product. Join the developer preview to shape it.

Coming

A typed SDK

A first-party client for TypeScript and Python, so calling the API and streaming agent activity is one install, not hand-rolled HTTP.

Coming

API keys for your apps

Scoped, revocable keys for server-to-server access, so an external app can act as itself, alongside today's session auth.

Coming

Outbound event webhooks

Subscribe your own URL to fleet events, an approval resolved, a task done, a run finished, and react in your stack.

Coming

Pluggable model endpoints

Point an agent at any OpenAI-compatible endpoint, managed or local, and keep the same coordination layer around it.

Extensibility

Your tools, in your agents' hands.

The cleanest way to extend Nomik isn't a plugin format we invented, it's the one your agents already use. Mount an MCP server and its tools show up next to every connected integration, gated by the same risk model.

  • Standard protocol: any MCP server works, the ones you find and the ones you write.
  • Same guardrails: your tools inherit the approval gates and the audit trail.
  • No raw secrets: agents act through scoped access, never your credentials in the clear.
