Architecture Deep-Dive

Decision p50 ≤100ms; rail latency per scheme

Runtime Path:

  1. Agent → /payments.intent (idempotent)
  2. Policy Engine → evaluate (context, identity, history)
  3. Decision: allow | require-approval | deny
  4. If allow → Router selects rail → partner executes
  5. Events: payment.approved, payment.settled, policy.triggered
SDK → API → AuthN/Z → Policy Eval → Router → Partner PSP/Bank → Callback/Webhook
                      ↘ Logs (WORM store) ↙ Alerts (Slack/Email/SIEM)

KYA — Know Your Agent™

First-class agent identity and delegation framework

{
  "agent_id": "did:ai:inventory_bot",
  "principal": "kyb:acme_inc",
  "scopes": ["payments:send", "payments:simulate"],
  "limits": {"daily_eur": 10000, "per_txn_eur": 2500},
  "allowlist": ["iban:DE89370400440532013000"],
  "revocation": {"mode": "immediate"},
  "credentials": [{"type": "VerifiableCredential", "issuer": "acme"}]
}

Lifecycle: register → attest → monitor → revoke → re-issue

Signals: velocity, purpose match, recipient history, geofence, time windows

Policy-as-Code

Express compliance as executable rules

version: 1
policy:
  name: procurement_daily_cap
  applies_to: did:ai:inventory_bot
  limits:
    per_txn: { amount: 2500, currency: EUR }
    daily:   { amount: 10000, currency: EUR }
  recipients:
    allow: ["iban:DE89...3000", "iban:FR76...0000"]
  approvals:
    required_if:
      - condition: amount > 2000 EUR
        approvers: ["role:cfo", "role:ops_manager"]
  sanctions_check: true
  alerts:
    channels: ["slack:#payments", "email:finops@acme.com"]
{"limits":{"per_txn":2500,"daily":10000},"currency":"EUR","approvals":{"threshold":2000}}

API Endpoints (v1)

RESTful API with idempotency and webhooks

Request Example:

{
  "agent_id":"did:ai:inventory_bot",
  "amount":12000,
  "currency":"EUR",
  "recipient":"iban:FR76...",
  "rail":"sepa_instant",
  "purpose":"restock_trigger",
  "idempotency_key":"uuid"
}

Response Example:

{
  "status":"approved",
  "transaction_id":"txn_01H...",
  "rail":"sepa_instant",
  "policy_decision":"auto_approved",
  "eta":"instant"
}

Errors: POLICY_DENIED, APPROVAL_REQUIRED, RAIL_UNAVAILABLE, SANCTIONS_MATCH, LIMIT_EXCEEDED

Webhooks: policy.alert, payment.approved, payment.failed, payment.settled (HMAC-signed)
