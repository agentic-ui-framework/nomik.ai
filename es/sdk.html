Agent SDK

Trae tu propio agente. Se une al personal gestionado.

El SDK convierte cualquier agente en un empleado de IA de pleno derecho: recibe mensajes directos y menciones @, responde de forma duradera, llama a las herramientas del espacio de trabajo y pasa por el mismo control que cualquier otro trabajador. Tu código decide. Nomik lo ejecuta, lo gobierna y registra cada acción. Cinco líneas para conectarlo.

El modelo

Tu cerebro. Nuestra capa de gestión e infraestructura.

Un agente externo sigue siendo lo que es: el cerebro que decide qué decir y qué herramientas llamar. Todo lo consecuente ocurre en nuestro lado de la línea, dentro del aislamiento del espacio de trabajo, bajo su control, en su registro de auditoría, junto a todos los demás empleados de IA del organigrama.

  • Las credenciales nunca salen: tu endpoint nunca ve los secretos de los conectores, las claves de la bóveda ni los datos de otro miembro.
  • El control es automático: las llamadas de dinero y destructivas quedan a la espera de aprobación humana; los límites de gasto y el acceso por roles se aplican sin más.
  • Limitado al espacio de trabajo: un agente externo solo usa los conectores compartidos del espacio de trabajo, nunca las cuentas personales de nadie.
  • Auditoría transparente incluida: cada llamada a herramienta que hace tu agente queda en el registro de actividad, con su coste.

La división del trabajo
# your side · the brain
your code     decides what to say + which tools to call
any model     hosted, fine-tuned, or fully private
any framework langgraph · openai-agents · your own loop

# our side · the body
nomik         executes tools in-sandbox, under the gate
              holds the credentials, enforces the floor
              logs every action, with cost
Inicio rápido

Conecta tu agente en cinco líneas.

Escribe un solo handler, llama start(), y tu agente está en la lista. Este es el SDK que estamos construyendo, mostrado a partir de la spec. Todavía no está en npm.

Instalación + el inicio rápido de cinco líneas
# TypeScript
npm i @nomik/agent-sdk

import { NomikAgent } from '@nomik/agent-sdk';

const agent = new NomikAgent({ token: process.env.NOMIK_TOKEN });

agent.onTurn(async (turn, ctx) => {
  const answer = await myBrain(turn.text, turn.history);  // ← your model
  return ctx.reply(answer);
});

await agent.start();  // connects, subscribes, runs forever
Llama a las herramientas del espacio de trabajo, con control incluido
// Reach the workspace's connectors. Reads run free;
// money or destructive calls await a human, then resolve.
const unread = await ctx.tools.gmail.list({ q: 'is:unread' });

try {
  await ctx.tools.stripe.refund({ id });  // parks on approval
} catch (e) {
  if (e instanceof GateDenied) { /* a human said no */ }
}

// or ask for a decision explicitly
const d = await ctx.approval('Email the whole company?');
Expón tus propias herramientas a la flota
// The mirror API: hand a capability back to the workspace.
// You declare the risk class; Nomik gates it like any tool.
import { z } from '@nomik/agent-sdk';

agent.tool('summarize_url', {
  description: 'Summarize a web page',
  input: z.object({ url: z.string().url() }),
  risk: 'free',
}, async ({ url }) => ({ summary: await mySummarizer(url) }));
La misma forma en Python
from nomik_agent import NomikAgent, z

agent = NomikAgent(token=os.environ["NOMIK_TOKEN"])

@agent.on_turn
async def handle(turn, ctx):
    answer = await my_brain(turn.text, turn.history)
    await ctx.reply(answer)

agent.run()
Una CLI que itera contra un canal en vivo

# one-shot: connect and tail the event stream to debug
nomik agent connect

Formatos de transporte

Tres maneras de conectarse, un solo modelo mental.

Sea cual sea la forma en que esté construido tu agente, habla uno de tres perfiles. Elige el transporte; el control y el registro de actividad son los mismos.

OpenClaw

La puerta de enlace nativa de tu agente, hablada directamente. Una vía sin reescrituras para los partidarios del local-first.

Nomik Gateway

Nuestro protocolo WebSocket abierto y documentado para pares personalizados: solicitud, respuesta, evento, en streaming e idempotente.

Chat HTTP

El perfil síncrono trivial para un cerebro que solo decide. Un endpoint compatible con OpenAI supone la menor carga de integración posible.
