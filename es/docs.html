Primeros pasos

Nomik es un equipo de agentes de IA que controlas desde el chat. Para los desarrolladores, también es una plataforma: una API REST para controlar espacios de trabajo y agentes, una superficie de herramientas MCP que tus agentes hablan de forma nativa y un modelo de descriptores para añadir integraciones. Construyes en torno a las mismas primitivas que usa la aplicación.

La URL base de la API es https://api.nomik.ai/v1. Todo devuelve el mismo envoltorio, así que una respuesta siempre incluye success más data más metadata (que lleva un requestId para soporte y trazabilidad).

Autenticación

Las llamadas se autentican con un token bearer en el encabezado Authorization. Los tokens se firman y se verifican contra un conjunto de claves públicas, y llevan el principal que realiza la solicitud.

Encabezado Authorization
curl https://api.nomik.ai/v1/me \
  -H "Authorization: Bearer $TOKEN"

Próximamente: claves de API revocables y con alcance acotado para el acceso de servidor a servidor, de modo que una aplicación externa pueda autenticarse como ella misma en lugar de como una sesión de usuario.

Referencia de la API

Los recursos principales reflejan el producto: los espacios de trabajo, los agentes que contienen, las ejecuciones que esos agentes realizan y la actividad que emiten. Algunas rutas representativas:

Rutas representativas
GET  /v1/workspaces/:ws/agents          # list agents
POST /v1/workspaces/:ws/agents          # create an agent
POST /v1/agents/:id/runs                # start a run
GET  /v1/workspaces/:ws/activity        # read the activity stream

Envoltorio de respuesta

El éxito y el fallo comparten una misma forma, así que analizas una sola vez. Los errores llevan un code legible por máquina junto a un mensaje legible para humanos.

Forma del error
{
  "success": false,
  "error": { "code": "workspace_required",
             "message": "No workspace in context." },
  "metadata": { "requestId": "req_3a9" }
}

MCP & herramientas

Los agentes actúan a través de herramientas y hablan el Model Context Protocol. Cada integración conectada aparece como herramientas MCP; y también todo lo que montes tú mismo. Para ampliar un agente, no aprendes un formato de plugin a medida: lo apuntas a un servidor MCP.

Monta tu propio servidor MCP
{
  "mcpServers": {
    "warehouse": {
      "command": "npx",
      "args": ["-y", "@acme/warehouse-mcp"]
    }
  }
}

Tus herramientas heredan el mismo control de riesgos y el mismo registro de auditoría que las integradas: las lecturas se ejecutan libremente, y las escrituras que mueven dinero o no se pueden deshacer esperan aprobación.

Conectores

Un conector es un descriptor, no un servicio que tengas que escribir. Declara la URL base del proveedor, cómo se autentica y la clase de riesgo de sus acciones; la fábrica lo convierte en herramientas reales y sujetas a aprobación, y guarda la credencial en la bóveda propia de cada espacio de trabajo.

Un descriptor de conector
export const acme = {
  kind: "acme",
  displayName: "Acme",
  auth: "api_key",         // or "oauth"
  baseURL: "https://api.acme.com/v2",
  risk: "outbound",        // money | destructive | outbound | data | …
}

El riesgo es el contrato: decide qué llamadas se ejecutan en silencio y cuáles se pausan para un humano. Las lecturas en cualquier conector siempre se ejecutan libremente.

Webhooks

Apunta el webhook de un proveedor a tu espacio de trabajo y los eventos entrantes se convierten en trabajo. Cada entrega se verifica por firma, se clasifica por tipo y se enruta al equipo que la gestiona.

Endpoint de webhook entrante
POST /v1/webhooks/:workspace/:kind/:binding

# verified → classified → published to the fleet
# e.g. a Stripe dispute wakes the finance squad

Próximamente: webhooks de eventos salientes; suscribe tu propia URL a eventos de la flota, como una aprobación resuelta o una ejecución finalizada.

Despliegue

Nomik funciona de tres maneras: como SaaS gestionado, en tu propia nube privada o totalmente en local (on-prem) con modelos locales y sin que nada salga de tu perímetro. La API y la superficie de herramientas son las mismas en los tres casos.
