Prise en main

Nomik est une main-d'œuvre IA que vous pilotez depuis le chat. Pour les développeurs, c'est aussi une plateforme : une API REST pour piloter les espaces de travail et les agents, une surface d'outils MCP que vos agents parlent nativement, et un modèle de descripteurs pour ajouter des intégrations. Vous construisez autour des mêmes primitives que l'application.

L'URL de base de l'API est https://api.nomik.ai/v1. Tout renvoie la même enveloppe : une réponse contient toujours success plus data plus metadata (qui porte un requestId pour le support et la traçabilité).

L'accès est aujourd'hui réservé à l'aperçu développeur. Demander un accès →

Authentification

Les appels sont authentifiés par un jeton bearer dans l'en-tête Authorization. Les jetons sont signés et vérifiés à l'aide d'un jeu de clés publiques, et portent le principal à l'origine de la requête.

En-tête Authorization
curl https://api.nomik.ai/v1/me \
  -H "Authorization: Bearer $TOKEN"

Bientôt : des clés d'API à portée limitée et révocables pour l'accès serveur à serveur, afin qu'une application externe puisse s'authentifier en tant qu'elle-même plutôt qu'en tant que session utilisateur.

Référence de l'API

Les ressources principales reflètent le produit : les espaces de travail, les agents qu'ils contiennent, les exécutions que ces agents réalisent et l'activité qu'ils émettent. Quelques routes représentatives :

Routes représentatives
GET  /v1/workspaces/:ws/agents          # list agents
POST /v1/workspaces/:ws/agents          # create an agent
POST /v1/agents/:id/runs                # start a run
GET  /v1/workspaces/:ws/activity        # read the activity stream

Enveloppe de réponse

Le succès et l'échec partagent la même forme : vous n'analysez qu'une fois. Les erreurs portent un code lisible par machine accompagné d'un message humain.

Forme d'erreur
{
  "success": false,
  "error": { "code": "workspace_required",
             "message": "No workspace in context." },
  "metadata": { "requestId": "req_3a9" }
}

MCP & outils

Les agents agissent par le biais d'outils, et ils parlent le Model Context Protocol. Chaque intégration connectée apparaît sous forme d'outils MCP ; il en va de même pour tout ce que vous montez vous-même. Pour étendre un agent, vous n'apprenez pas un format de plugin sur mesure : vous le pointez vers un serveur MCP.

Montez votre propre serveur MCP
{
  "mcpServers": {
    "warehouse": {
      "command": "npx",
      "args": ["-y", "@acme/warehouse-mcp"]
    }
  }
}

Vos outils héritent de la même validation des risques et du même journal d'audit que les outils intégrés : les lectures s'exécutent librement, les écritures qui déplacent de l'argent ou sont irréversibles attendent une approbation.

Connecteurs

Un connecteur est un descripteur, pas un service que vous devez écrire. Déclarez l'URL de base du fournisseur, sa méthode d'authentification et la classe de risque de ses actions ; la fabrique en fait de véritables outils soumis à validation et stocke l'identifiant dans le coffre-fort propre à chaque espace de travail.

Un descripteur de connecteur
export const acme = {
  kind: "acme",
  displayName: "Acme",
  auth: "api_key",         // or "oauth"
  baseURL: "https://api.acme.com/v2",
  risk: "outbound",        // money | destructive | outbound | data | …
}

Le risque est le contrat : il décide quels appels s'exécutent silencieusement et lesquels s'interrompent pour un humain. Les lectures sur n'importe quel connecteur s'exécutent toujours librement.

Webhooks

Dirigez le webhook d'un fournisseur vers votre espace de travail et les événements entrants deviennent du travail. Chaque livraison est vérifiée par signature, classée par type et acheminée vers les employés IA qui la traitent.

Point de terminaison de webhook entrant
POST /v1/webhooks/:workspace/:kind/:binding

# verified → classified → published to the fleet
# e.g. a Stripe dispute wakes the finance department

Bientôt : des webhooks d'événements sortants, abonnez votre propre URL aux événements de la flotte comme une approbation résolue ou une exécution terminée.

Déploiement
