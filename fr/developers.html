En pratique

Un aperçu de ce à quoi cela ressemble.

Du REST propre, une configuration MCP familière et un connecteur en une ligne. Tous les détails, l'authentification et les codes d'erreur se trouvent dans la documentation.

REST · lister les agents d'un espace de travail
# Authenticated with a bearer token
curl https://api.nomik.ai/v1/workspaces/$WS/agents \
  -H "Authorization: Bearer $TOKEN"

# → every response shares one envelope
{
  "success": true,
  "data": [
    { "id": "agt_8f2", "name": "Fin", "role": "finance" }
  ],
  "metadata": { "requestId": "req_3a9" }
}
MCP · donner aux agents un outil que vous hébergez
// Mount any MCP server into the workspace.
// Its tools join the ones agents already have.
{
  "mcpServers": {
    "warehouse": {
      "command": "npx",
      "args": ["-y", "@acme/warehouse-mcp"],
      "env": { "REGION": "eu" }
    }
  }
}
Connecteur · une ligne devient une vraie intégration
// No SDK to write. Declare it; the factory builds the actions,
// the vault holds the credential, and risk gates the writes.
export const acme = {
  kind: "acme",
  displayName: "Acme",
  category: "Operations",
  auth: "api_key",            // or "oauth"
  baseURL: "https://api.acme.com/v2",
  risk: "outbound",           // reads stay free; writes wait for approval
}

Nomik Desktop

Le même SDK alimente un agent local-first.

Nomik Desktop est un agent autonome qui s'exécute sur votre machine, avec votre modèle et vos outils, sûr par défaut. Une connect et il se connecte à un espace de travail Nomik via ce SDK même : mains locales, cerveau dans l'espace de travail, le tout sous la validation humaine.

  • Sûr par défaut : secrets chiffrés, exécution priorisant le bac à sable, outils soumis à validation selon le risque, un journal d'audit, le tout actif dès l'installation.
  • Natif de l'espace de travail : encadré par la même validation, les mêmes plafonds de dépenses, le même RBAC et le même journal d'activité que votre flotte utilise déjà.
  • Apportez vos compétences : importez vos compétences OpenClaw existantes, puis versionnez-les et évaluez-les.
